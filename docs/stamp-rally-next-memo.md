# スタンプラリー 次回変更メモ

作成日: 2026-06-21

## 次回以降に変更すること

- ユーザー登録はQRからだけではなく、URLからも事前に登録できるようにする。
  - 現状: 会場QR読み取りを起点に、初回ニックネーム登録して参加者作成。
  - 変更後イメージ: 事前共有URLから先にユーザー登録でき、当日はQR読み取りだけでスタンプ取得できる。
- プロジェクトごとに表示画面を変える。
  - フェス、連続ライブ、単発イベントなど、プロジェクト単位でUI/文言/導線を変えられるようにしたい。
- ユーザー情報の紐付けを検討する。
  - 現状はニックネーム＋端末ローカル＋復元コードが中心。
  - 今後は個人情報、外部ID、プロジェクト参加情報との紐付け方法を決める。

## ともやに聞きたいこと

> ともや = 今回の発注先

### 1. 個人情報の紐付け方法

現状はニックネーム登録のみなので、ユーザーを個人情報に紐付けたい。
方法としてどちらがよさそうか相談する。

1. メールアドレス紐付け
2. LINE紐付け

確認したい観点:
- ユーザーにとって登録が面倒すぎないか
- 機種変更・端末変更時の復元がしやすいか
- なりすまし防止になるか
- 運営側が取得・管理する個人情報として重くなりすぎないか
- Supabase/Auth/LINEログイン等の実装コスト

### 2. プロジェクトごとの表示画面変更とユーザー登録の紐付け

プロジェクトごとに表示画面を変える場合、ユーザー登録もプロジェクトごとに紐付けるべきか相談する。

確認したいこと:
- 1人のユーザーが複数プロジェクトに参加する前提にするか
- ユーザー情報は全プロジェクト共通にするか、プロジェクト単位で別管理にするか
- プロジェクトごとに必要な登録項目を変えられるようにするか
- 表示画面だけ変えるのか、ユーザーフローや登録項目まで変えるのか

## 未整理メモ

- QR起点だけだと、当日現地で登録が発生して少し手間になる。
- 事前登録URLがあれば、SNSや公式サイトから先に登録してもらえる。
- 個人情報を持つ場合は、便利になる一方で管理責任も増える。
- LINE連携はユーザーにとって楽そうだが、実装・審査・運用面を確認したい。
- メール紐付けは実装しやすそうだが、現場で入力が面倒になる可能性がある。

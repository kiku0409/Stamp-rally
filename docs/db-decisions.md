# DB・インフラ関連の意思決定メモ

## プロジェクト概要
- 10月開催のストリートライブ×飲食店コラボフェスのスタンプラリーアプリ
- 会場・お店を回遊してもらうためのデジタルスタンプ
- 顧客情報（特に年代）の取得も目的のひとつ

---

## Supabaseを選択した理由（仕様書スライド用）

1. **コストゼロで始められる**
   無料プランで参加者1万人・イベント100会場分まで対応可能。初期フェーズの費用リスクがない。

2. **登録・セットアップが簡単**
   AWSのような専任エンジニア不要。ダッシュボードのUIが直感的で、非エンジニアでも管理できる。

3. **顧客データをすぐExcelで分析できる**
   管理画面からCSVを1クリックでエクスポート可能。年代別・会場別の集計をExcelで即実施できる。

4. **セキュリティが標準装備**
   RLS（行レベルセキュリティ）により、参加者データを安全に保護できる。

5. **将来の拡張に対応できる**
   規模が拡大しても$25/月のProプランへ移行するだけでそのまま使い続けられる。LINEログイン連携も対応。

6. **他DBとの比較で最適**
   AWS（高コスト・複雑）、Firebase（コード全面書き直し必要）と比較して、現フェーズに最も適している。

## Supabase 無料プランの制限と実態

| 項目 | 無料プランの上限 |
|---|---|
| DBストレージ | 500MB |
| 同時接続数 | 200 |
| 月額料金 | $0 |
| 注意点 | **1週間アクセスがないとプロジェクトが停止する** |

### 無料枠で入るデータ量（試算）

1レコードあたりのサイズ：
- participants（参加者）：約150バイト/人
- events（イベント）：約500バイト/件
- event_stamps（スタンプ記録）：約120バイト/件

| シナリオ | 参加者数 | イベント数 | 使用容量 |
|---|---|---|---|
| 今回のフェス | 100人 | 20会場 | 約1MB |
| 中規模 | 1,000人 | 50会場 | 約20MB |
| 上限目安 | **約1万人** | **100会場** | 約500MB |

**→ 参加者1万人・イベント100会場まで無料で対応可能。有料化が必要になるのは参加者1万人超から。**

### 本番前にやること
- イベント直前に **Pro プラン（$25/月）へ昇格** するか、定期アクセスで停止を防ぐ

## 他DBとの比較（検討・却下済み）

| サービス | 月額 | 結論 |
|---|---|---|
| **Supabase Free**（採用） | $0 | ◎ 最適 |
| Supabase Pro | $25/月 | 本番規模になったら移行 |
| AWS RDS | $15〜30/月 | 過剰・複雑・高コスト → 却下 |
| AWS DynamoDB | $0〜従量 | 設定複雑・専門知識必要 → 却下 |
| Neon | $0 | Supabaseとほぼ同等だが管理UIが弱い → 却下 |
| Firebase Firestore | $0〜 | コードの大規模書き直しが必要 → 却下 |
| PlanetScale | — | 2024年に無料プラン廃止 → 却下 |
| Excel | — | DBにはなれない。分析ツールとして使う |

### AWSについて
大企業向けで運用・設定・料金管理が複雑。エンジニア常駐が前提の設計のため、現フェーズでは費用対効果が悪い。数万人規模になったら検討。

### Excelについて
DBの代替にはならないが、Supabaseの管理画面からCSVを1クリックでエクスポートして、Excel で年代別集計・来場者分析を行うのはベストな使い方。

## 顧客情報取得の方針

### 課題
- ストリートライブ系は顧客情報がチケットでしか把握できない
- 将来の広告・協賛に向けて年代情報が必須
- 登録をいかにスムーズにするかが鍵

### 対応済み（2026-06-16実装）
- `participants` テーブルに `birth_decade` カラムを追加
- 登録フォームに年代ボタン（10代 / 20代 / 30代 / 40代 / 50代以上）を追加
- ニックネーム＋年代の両方を入力しないと登録できない仕様

### 将来の拡張案
- **LINEログイン**（Supabase Auth連携）：無料。取得できるのは表示名・LINE UIDのみ（年齢・メアドは取れない）
- **LINE公式アカウント＋Messaging API**：月200通まで無料。スタンプ取得後に友だち追加を促して特典配信が可能

## マイグレーション

既存DBに年代カラムを追加する場合は以下を Supabase SQL Editor で実行：

```sql
ALTER TABLE participants ADD COLUMN IF NOT EXISTS birth_decade TEXT;
```

ファイル：`supabase/migrations/add_birth_decade.sql`

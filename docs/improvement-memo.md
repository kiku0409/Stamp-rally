## A. データ保全

### A-1. スタンプ取得データを機能改修で消さない
- **現状**: 来場者データは localStorage の `participant_id` + `recovery_code` で識別。DBのデータは消えないが、マイグレーションのカスケード削除（tier削除など）で `participant_rewards` が消える既知リスクあり（HANDOFF §3-3）。
- **要望**: 改修のたびにユーザーのスタンプ・特典データが失われないようにしたい。
- **検討事項**:
  - マイグレーション方針の明文化（削除系DDLは慎重に）
  - `participants` / `event_stamps` / `participant_rewards` に論理削除フラグを検討
  - 改修前後のデータ確認手順をドキュメント化

## D. 管理者画面の構成変更

### D-1. プロジェクト詳細ページを4タブに分割
- **現状**: プロジェクト詳細1ページにイベント・特典・取得者・メンバー・削除・再申請が全部集まっていて縦に長い。
- **要望**: 以下の4ページ（またはタブ）に分割
  1. **イベント一覧ページ** — イベントの追加・編集・削除
  2. **特典管理ページ** — 特典段階の設定
  3. **スタンプ取得者・特典取得者管理ページ** — 取得者一覧・CSV書き出し
  4. **メンバー・共同編集者ページ** — メンバー一覧・招待・参加コード表示
- **スコープ**: `app/admin/projects/[id]/page.tsx` の分割。サイドナビまたはタブUIを追加。

### D-2. イベントが増えるとスクロールが多い（ユーザー画面）
- **現状**: スタンプ帳はプロジェクト単位でイベントを縦並びに表示。
- **検討事項**:
  - カード折りたたみ / 横スクロール / ページング
  - プロジェクト絞り込みフィルタ

## E. 認証・セッション

### E-1. 管理者が再ログインを求められる頻度を減らしたい
- **現状の仕組み**:
  - Supabase Auth のアクセストークン有効期限: **1時間**
  - リフレッシュトークン: 最大 **60日**（タブを開き続けていれば自動更新される）
  - `AdminLayout.tsx:16` で `getSession()` を毎ページ呼び出し。セッションが null なら `/admin/login` へリダイレクト。
  - **つまり**: タブを閉じて60日以上放置、またはブラウザのストレージをクリアするとログアウト状態になる。通常の使い方（同じブラウザで継続利用）では再ログインはほぼ不要なはず。
- **要望の確認が必要**: 具体的にどのタイミングで再ログインが発生しているか教えてもらえると原因を特定しやすい（例: スマホでSafariを使っている場合は ITP によって7日でストレージが消える）。
- **対策候補**:
  - iOS Safari の場合は PWA としてホーム画面追加でストレージ保持
  - `supabase.auth.onAuthStateChange` でリフレッシュを確実に行う実装の確認

### B-4. 特典チケットタップ時にポップアップ表示
- **要望**: スタンプ帳で獲得した特典チケットをタップしたとき、スタンプ取得時と同じようなフルスクリーンポップアップ（モーダル）を表示したい。
- **現状**: `RewardTicketModal` が既にあるが、スタンプ帳からの動線・演出が弱い可能性あり。要確認。
- **実装方針**: スタンプ帳の特典チケット一覧タップ → `RewardTicketModal` をフルスクリーンで表示。スタンプ取得演出（`/event/[qr_token]/stamp`）のアニメーション参考に。

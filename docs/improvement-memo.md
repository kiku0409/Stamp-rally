## E. 認証・セッション

### E-1. 管理者が再ログインを求められる頻度を減らしたい
- **現状の仕組み**:
  - Supabase Auth のアクセストークン有効期限: **1時間**
  - リフレッシュトークン: 最大 **60日**（タブを開き続けていれば自動更新される）
  - `AdminLayout.tsx:16` で `getSession()` を毎ページ呼び出し。セッションが null なら `/admin/login` へリダイレクト。
  - **つまり**: タブを閉じて60日以上放置、またはブラウザのストレージをクリアするとログアウト状態になる。通常の使い方（同じブラウザで継続利用）では再ログインはほぼ不要なはず。
- **要望の確認が必要**: 具体的にどのタイミングで再ログインが発生しているか教えてもらえると原因を特定しやすい（例: スマホでSafariを使っている場合は ITP によって7日でストレージが消える）。
- **対策候補**:
  - iOS Safari の場合は PWA としてホーム画面追加でストレージ保持
  - `supabase.auth.onAuthStateChange` でリフレッシュを確実に行う実装の確認
